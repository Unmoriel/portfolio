Bot Discord : LoLcam

Un bot discord qui se connect à l'API de Riot Games pour envoyer des résumés des dernières parties des joueurs inscrit sur le serveur.

Voir le Github

Asynchrone

Toute les fonctions du bot sont définis comme asynchrone pour ne pas qu'elle se bloques entre elles

Bot Public

Le bot est capable de gérer plusieurs serveurs

Base de donnée Mariadb

Enfin, toutes les données nécessaires au fonctionnement du bot (pseudo, nom des serveurs, etc) sont sauvegardé sur une base de données MariaDB

Pourquoi ce projet ?

J'ai commencé ce projet pour travailler sur communication entre différentes API. Ce bot a beaucoup amusé mes amis et j'ai donc continué à le développer.
J'ai commencé à faire une première version qui stockait toutes les données dans des fichiers JSON, le code était sur un seul fichier et le bot pouvais gérer qu'un seul serveur discord. J'ai pus experimenter les limites d'un projet mal géré et c'est pour cela que j'ai décidé de développer une deuxième version du bot pouvant qui respecte les bases SOLID et DRY tout en stockant les données sur une base de donnée MariaDB.

Déroulement du projet :

Ce projet a été réalisé en parallèle de mes études. J'ai donc commencé par réalisé une première version assez basique mais fonctionnelle pour un serveur discord. Cette version a posé les bases avec les appels à l'API de Riot Games, la création de l'image avec tous les personnages de la partie, son stockage sur cloudinary et les "embed" (messages stylisés de discord). J'ai ensuite développé une deuxième version du bot, avec une base de donnée MariaDB pour stocker les données des joueurs et des serveurs. J'ai aussi ajouté la possibilité de gérer plusieurs serveurs discord et j'ai découpé les parties du code. Aujourd'hui le bot est désactivé le temps que je reprenne le développement.
